OWASP and LLM/Generative AI Security

Authors: OWASP

This document excerpt focuses on the extensive resources and frameworks provided by the Open Web Application Security Project (OWASP) to address the unique security challenges posed by Large Language Models (LLMs) and Generative AI. It highlights the broader OWASP LLM and Generative AI Security Project, which offers comprehensive guidance on mitigating risks and establishing secure development practices. The excerpt emphasizes the value of OWASP's existing projects, such as the Application Security Verification Standard (ASVS), Testing Guide, Web Security Testing Guide (WSTG), Cheat Sheet Series, Threat Modeling Project, and Threat Dragon tool, which are directly applicable to LLM-based systems. Specifically, the ASVS provides a framework for verifying the security of applications integrating LLMs, while the Testing Guide and WSTG offer methodologies for testing. The Cheat Sheet Series provides concise guidance on specific security techniques, and the Threat Modeling Project, combined with the Threat Dragon tool, enables effective threat analysis. Crucially, the document stresses the importance of supply chain security, recommending the use of OWASP Dependency-Check and Dependency-Track to manage dependencies. Beyond technical security, the excerpt advocates for a holistic approach, suggesting the application of the OWASP Software Assurance Maturity Model (SAMM) to develop robust software security programs and the OWASP DevSecOps Maturity Model (DSOMM) to integrate security into operational aspects of LLM infrastructure and data management. The overall message is that OWASP's established projects and tools provide a valuable and transferable foundation for securing LLM deployments, alongside broader considerations for data privacy, ethical implications, and responsible AI development.
