Context Engineering for Agents

Authors: Andrej Karpathy

This document explores the concept of 'Context Engineering' within the context of Large Language Models (LLMs) and their application to agent development. It frames LLMs as analogous to operating systems, with the context window acting as RAM ‚Äì a limited working memory. The core idea is that effective LLM applications require careful management of context, much like optimizing RAM usage in a computer. The document highlights several key context types that need to be considered: Instructions (prompts, memories, few-shot examples, tool descriptions), Knowledge (facts, memories), and Tools (feedback from tool calls). A significant focus is placed on the rise of 'agents' ‚Äì systems that interleave LLM invocations with tool calls, particularly for complex, long-running tasks. These agents accumulate a large number of tokens through repeated tool calls and LLM interactions, which can quickly exceed the context window's capacity. This presents a significant challenge, necessitating a strategic approach to 'context engineering' to efficiently manage and prioritize the information used by the agent. The document implicitly suggests that techniques for compressing and isolating context will be crucial for successful agent development, aligning with the broader need to maximize the utility of the LLM's context window. The document draws a parallel to operating system RAM management, emphasizing the importance of curation and optimization within the LLM's context window. It sets the stage for a deeper dive into strategies for handling the increased complexity introduced by agents and their reliance on tool calls.
