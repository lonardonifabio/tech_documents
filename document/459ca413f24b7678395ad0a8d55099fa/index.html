Rules of Machine Learning: Best Practices for ML Engineering

Authors: Martin Zinkevich

This document provides a set of best practices for machine learning engineering, drawing from Google's internal guidelines and aiming to assist those with a foundational understanding of machine learning. It establishes a style for ML development, mirroring the Google C++ Style Guide, and emphasizes practical application. The core principles revolve around a pragmatic approach, advocating for launching products without relying solely on machine learning, prioritizing metrics design and implementation, and selecting machine learning solutions over complex heuristics. The document outlines key phases, starting with 'ML Phase I: Your First Pipeline,' which stresses simplicity, infrastructure validation, and careful handling of data pipelines to avoid dropped data. A significant portion focuses on monitoring and alerting, emphasizing the importance of understanding freshness requirements, proactively detecting problems, and watching for silent failures. Furthermore, it highlights the need for feature set ownership and comprehensive documentation. Regarding objectives, the document advises against overthinking and instead recommends choosing simple, observable, and attributable metrics for initial optimization. The overall tone is practical and actionable, providing a framework for building robust and maintainable machine learning systems.
