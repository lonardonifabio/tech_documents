How to Avoid Machine Learning Pitfalls: A Guide for Academic Researchers

Authors: Michael A. Lones

This document provides a concise guide specifically tailored for academic researchers seeking to avoid common pitfalls when utilizing machine learning techniques. Primarily aimed at research students and newcomers to the field, it outlines key considerations across five stages of the machine learning process: pre-model building, reliable model construction, robust model evaluation, fair model comparison, and appropriate result reporting. The guide emphasizes the importance of rigorous comparisons and valid conclusions, reflecting the scholarly concerns prevalent in academic research. Key Concepts:

  • Avoiding machine learning pitfalls in academic research
  • Rigorous comparisons and valid conclusions
  • Five stages of the machine learning process
  • Class imbalance refers to situations where some classes have significantly fewer samples than others.
  • Complex ML models (e.g., deep neural networks) can overfit small datasets.
  • Collaboration with domain experts is crucial for problem definition, feature selection, and model appropriateness.
  • Independence of test set: The test set should be independent from the training data to accurately assess model generality.
  • Overlearning: A model can overlearn specific characteristics of the training data, leading to poor generalization.
  • Validation set usage: Using a separate validation set is crucial for evaluating model performance and preventing overlearning.
  • Early stopping: A technique to halt training based on performance on the validation set.
